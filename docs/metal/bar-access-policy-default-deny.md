# BAR Access Policy: Default Deny

## Purpose

Stage 7 adds a default-deny BAR access policy generator.

This stage converts the Stage 6 `bar-access-readiness-matrix.json` into a policy report that blocks all BAR/MMIO access.

The expected policy decision is:

    DENY_ALL_BAR_ACCESS

This is intentional.

The project is not ready for BAR mapping, MMIO reads, MMIO writes, PCI config-space writes, GPU reset logic, firmware loading, display initialization, framebuffer initialization, Metal acceleration, or DriverKit activation.

## Inputs

Required input:

    bar-access-readiness-matrix.json

Usually generated by:

    ./scripts/generate-bar-readiness-matrix.py ~/Desktop/H1mekaRTX-bar-inventory-test/bar-inventory-summary.json

## Usage

Generate the policy:

    ./scripts/generate-bar-access-policy.py ~/Desktop/H1mekaRTX-bar-inventory-test/bar-access-readiness-matrix.json

Optional output directory:

    ./scripts/generate-bar-access-policy.py ~/Desktop/H1mekaRTX-bar-inventory-test/bar-access-readiness-matrix.json --out-dir ~/Desktop/H1mekaRTX-bar-inventory-test

## Outputs

The script writes:

- `bar-access-policy.json`
- `bar-access-policy.md`

## Policy Mode

Current policy mode:

    DEFAULT_DENY

Effective decision:

    DENY_ALL_BAR_ACCESS

## BAR Slots

The generated policy covers:

- BAR0
- BAR1
- BAR2
- BAR3
- BAR4
- BAR5
- ROM

Every slot is marked:

- read: false
- write: false
- map: false
- poke: false

## Forbidden Operations

- PCI config-space writes
- MMIO reads
- MMIO writes
- BAR mapping
- BAR poking
- GPU reset logic
- firmware loading
- GSP initialization
- display engine initialization
- framebuffer initialization
- Metal acceleration attempts
- DriverKit activation

## Allowed Operations

- read existing Stage 3 text files
- read existing Stage 4 summary JSON
- read existing Stage 5 safety report
- read existing Stage 6 readiness matrix
- generate local JSON reports
- generate local Markdown reports

## Safety Boundary

This stage is read-only.

It only reads an existing Stage 6 JSON file and writes local report files.

It does not perform:

- ioreg collection
- system_profiler collection
- PCI config-space reads
- PCI config-space writes
- MMIO reads
- MMIO writes
- BAR memory mapping
- BAR memory poking
- GPU reset logic
- firmware loading
- GSP initialization
- display engine initialization
- framebuffer initialization
- Metal acceleration attempts
- DriverKit activation

## Notes

This stage intentionally blocks escalation.

A future stage may propose a review-only exception workflow, but this stage keeps the effective policy at `DENY_ALL_BAR_ACCESS`.
